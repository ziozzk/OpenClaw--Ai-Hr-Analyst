
## 飞书自动化流程

### 场景 1：新简历入库

**触发：** 收到简历文件
**动作：**
1. 解析简历内容
2. 在候选人表创建新记录
3. 创建评估报告文档
4. 发送通知给招聘负责人

### 场景 2：面试安排

**触发：** 候选人状态变更为"面试中"
**动作：**
1. 在面试记录表创建记录
2. 创建日历事件
3. 发送面试邀请给候选人和面试官
4. 准备面试评估文档

### 场景 3：面试完成

**触发：** 面试评估提交
**动作：**
1. 更新面试记录表
2. 计算平均分
3. 更新候选人状态
4. 通知下一步安排

### 场景 4：候选人对比

**触发：** 请求对比多个候选人
**动作：**
1. 从表格提取数据
2. 生成对比矩阵文档
3. 发送对比报告

### 场景 5：面试通知邮件发送 ⭐ NEW

**触发：** 用户发送面试通知指令
**动作：**
1. 从候选人表提取候选人信息（姓名、邮箱、岗位）
2. 从公司配置读取公司信息
3. 选择邮件模板（根据面试轮次）
4. 生成邮件草稿
5. 发送草稿给用户确认
6. 用户确认后发送邮件
7. 更新候选人状态为"面试中"
8. 在面试记录表创建记录
9. 发送飞书通知给 HR 团队

**触发命令：**
```
/hr 发送面试通知 [候选人]
/hr 安排面试 [候选人] [轮次] [时间]
/hr 加急面试 [候选人]
```

## 使用方式

### 在飞书中使用

1. **添加候选人** - 在多维表格中手动添加或发送简历给机器人
2. **查看看板** - 使用多维表格的看板视图查看招聘流程
3. **生成报告** - @机器人 请求生成评估报告
4. **对比候选人** - @机器人 请求对比指定候选人

### 命令示例

```
/hr 分析简历 [上传文件]
/hr 评估匹配度 [候选人] [岗位]
/hr 创建面试 [候选人] [轮次] [时间]
/hr 对比候选人 [候选人 A] [候选人 B] [候选人 C]
/hr 生成报告 [候选人]
/hr 导出看板

# 面试通知相关 ⭐ NEW
/hr 发送面试通知 [候选人姓名]
/hr 发送面试通知 排名第一的候选人
/hr 安排面试 陈琳 初试 下周一上午 10 点
/hr 加急面试 张伟
/hr 查看面试安排

```
