# 飞书面试通知功能集成方案

## 📋 功能概述

在飞书 Bot 中集成面试通知发送功能，支持：
- 飞书聊天命令触发
- 多维表格数据联动
- 邮件/飞书消息双通道通知
- 自动状态更新

---

## 🚀 飞书命令设计

### 主命令：`/hr`

```
/hr <子命令> [参数]
```

### 子命令列表

| 子命令 | 参数 | 说明 | 示例 |
|--------|------|------|------|
| `send_interview` | 候选人、轮次、时间 | 发送面试通知 | `/hr send_interview 陈琳 初试 下周一 10 点` |
| `schedule` | 候选人、轮次、时间 | 安排面试（内部） | `/hr schedule 张伟 复试 周三下午 2 点` |
| `urgent` | 候选人、时间 | 加急面试通知 | `/hr urgent 李悦 明天` |
| `status` | 候选人 | 查看面试状态 | `/hr status 陈琳` |
| `confirm` | 候选人、确认结果 | 确认面试 | `/hr confirm 陈琳 参加` |
| `reschedule` | 候选人、新时间 | 改期 | `/hr reschedule 陈琳 下周二 10 点` |

### 简化命令（自然语言）

```
发送面试通知给陈琳
安排陈琳面试，下周一上午 10 点
加急！给张伟发面试通知
陈琳确认参加面试
李悦需要改期，下周二可以吗
```

## 📊 多维表格联动

### 表结构更新

**候选人表新增字段：**
| 字段名 | 类型 | 说明 |
|--------|------|------|
| 面试通知发送时间 | 日期时间 | 首次发送时间 |
| 面试确认状态 | 单选 | 待确认/已确认/已拒绝/需改期 |
| 确认时间 | 日期时间 | 候选人确认时间 |
| 面试邮件链接 | 链接 | 邮件存档链接 |

**面试记录表新增字段：**
| 字段名 | 类型 | 说明 |
|--------|------|------|
| 通知方式 | 多选 | 邮件/飞书消息/短信 |
| 通知发送时间 | 日期时间 | |
| 确认截止时间 | 日期 | |
| 提醒次数 | 数字 | 已提醒次数 |

### 状态流转

```
待筛选 → 面试通知已发送 → 待确认 → 已确认 → 面试完成
                                    ↓
                                需改期 → 重新安排
                                    ↓
                                已拒绝 → 终止流程
```

## 📧 通知渠道

### 渠道 1：邮件通知

**适用场景：** 正式面试邀请、外部候选人

**配置：**
```yaml
邮件:
  SMTP:
    服务器：smtp.feishu.cn
    端口：587
    发件人：hr@company.com
  模板：templates/interview-email-templates.md
```

### 渠道 2：飞书消息

**适用场景：** 内部推荐、快速通知、已添加飞书好友的候选人

**消息格式：**
```
📧 面试邀请

尊敬的{姓名}：

您好！感谢您关注{公司}的{岗位}职位。
现邀请您参加面试。

📋 面试信息
• 时间：{日期} {时间}
• 形式：{现场/视频}
• 地点/链接：{地址或会议链接}

👥 联系人
• {HR 姓名}：{电话}

请回复确认是否参加。

{公司} 人力资源部
```

### 渠道选择逻辑

```
IF 候选人有飞书账号 AND 已添加好友
  → 发送飞书消息 + 邮件（双通道）
ELSE
  → 发送邮件
```

## ⏰ 提醒机制

### 自动提醒

| 时间点 | 动作 | 内容 |
|--------|------|------|
| 发送后 24h 未确认 | 第一次提醒 | "温馨提示：请确认面试安排" |
| 发送后 48h 未确认 | 第二次提醒 | "面试确认即将截止" |
| 截止前 2h | 最后提醒 | "面试确认今日截止" |
| 面试前 1 天 | 面试提醒 | "明日面试，请做好准备" |
| 面试前 2h | 再次提醒 | "面试即将开始" |

### 提醒配置

```yaml
提醒设置:
  启用：true
  渠道：飞书消息 + 邮件
  最大提醒次数：3
  提醒间隔：24 小时
  工作时间发送：true (9:00-18:00)
```

## 🔐 权限与安全

### 权限控制

| 操作 | 允许角色 |
|------|----------|
| 发送面试通知 | 招聘负责人、HR 管理员 |
| 确认面试 | 候选人本人 |
| 改期 | 招聘负责人、HR 管理员 |
| 取消面试 | HR 管理员 |

### 安全措施

- ✅ 邮件发送前必须用户确认
- ✅ 敏感信息脱敏（手机号、邮箱）
- ✅ 操作日志记录
- ✅ 飞书权限验证

## 📊 数据追踪

### 关键指标

| 指标 | 计算方式 | 目标 |
|------|----------|------|
| 通知发送数 | COUNT(面试通知) | - |
| 确认率 | 已确认/已发送 | >80% |
| 平均确认时间 | AVG(确认时间 - 发送时间) | <24h |
| 面试到场率 | 实际面试/已确认 | >90% |

## 🧪 测试计划

### 测试场景

1. **正常流程测试**
   - 发送面试通知 → 候选人确认 → 面试完成

2. **改期测试**
   - 发送通知 → 候选人要求改期 → 重新安排

3. **拒绝测试**
   - 发送通知 → 候选人拒绝 → 更新状态

4. **超时测试**
   - 发送通知 → 超时未确认 → 自动提醒

5. **边界测试**
   - 非工作时间发送
   - 候选人邮箱无效
   - 飞书消息发送失败
